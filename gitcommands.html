Git guide to commands

Note: This manual contains necessary commands for git when working on your own, contributing to other projects and working as a team

The Basics

Git push Git pull Git add (filename) Git status Git commit -m “first commit” These commands are used to add files to your repository through a terminal or git bash.git pull,pulls a repository straight from github onto your machine.To add files from your machine through a terminal or git bash we first use the command git add (filename). Then we use git commit -m “insert your own message based on the commit” to commit the file to a repo after that we check if it got committed or not by using git status to check if it worked if it’s green it’s ok if it’s red you need to go back or try committing again.After we committed and it shows the filename in green when using git status we use git push to push the file into the repo.

The Hard part

Part 1, Forking a repo. To fork some ones repo you first go onto github then we go to the top right where it should say fork we click that and we wait until it’s done. Once it’s done you should have a copy of that repo on your account. Part 2, Cloning the repo you forked. First you want to go onto your account and go to the forked repo you have, once you’re inside the repo you want to click the clone or download button (the green button at the top) if it’s not set to ssh (link to how to set up ssh keys if you haven’t already https://www.digitalocean.com/community/tutorials/how-to-set-up-ssh-keys--2) automatically change it to ssh from https and copy the the line that’s similar to this git@github.com:USERNAME/REPONAME.github.io.git We then go into the terminal/git bash and use the command git clone (git@github.com:USERNAME/REPONAME.github.io.git) It should have cloned now. Part 3, Creating a remote for the forked repo. Before we create a remote we want to check if there’s already a remote and to do that we use the command git remote -v and it should tells what remotes are there. To create a remote we also need to be in the location of the repo on your machine to get to it we use ls to check where we are and cd to go into folders through the terminal or git bash, once we are in the repo location we use git remote add (name of the remote) (git@github.com:USERNAME/REPONAME.github.io.git) And it should make that remote and to check if it did we use git remote -v, it should now have 4 remotes.

The Hard part 2

Part 4, Creating a branch. Creating a branch is necessary if we want to make a pull request to the person’s repo we forked so that we can add changes or fixes. Before we create a new branch we need to make sure we’re in the master branch To switch between branches we use git checkout (your branch name). To create a branch we use the command git checkout -b (name your branch) this should now create a new branch in the forked repo.Part 5, Pushing a change to a forked repo.To push a change to a forked repo we first use git add (filename) then we continue with git commit -m “your message” to commit it then use git status to check if the commit worked, then at the end while pushing we don’t use just use git push but instead we use git push (remote-name) (branch name) this should push to our branch in our forked repo. Then to create a pull request we go into the forked repo on your account and it should notify you at the top if you want to create a pull request and click the button, then type in what the request is about and what it changes or does and finally submit it. Generally when opening a pr we need a detailed description on it and a what,why or motivation factors to it as well. Part 6, Rebasing.Rebasing is necessary if some one already committed before you and the repo is ahead of you, rebasing basically pushes you ahead of a commit/commits of your branch.Git master Git pull origin master Git (branch name) Git rebase master Fasforward Filename… Fix up file Git add filename Git rebase continue Git push -f (remote) (branch)
